## Findings

### вЬЕ POSITIVE вАФ what we fixed

- **[POSITIVE] Date dedup now correct in both directions.** Before: "5 –Є—О–љ—П 2026" + "5 –Є—О–љ—П" вЖТ both kept (orphan partial). After: only full date kept. The fix added a reverse-substring check in `_add()`: if new phrase contains an existing shorter phrase, drop the old one. **Improves downstream `_match_in_text` precision** (no spurious "5 –Є—О–љ—П" claims).
- **[POSITIVE] Multi-word entity extraction now works.** Before: capitalized extractor only caught the first word of a sentence (length вЙ•6). "–Ь–Є–љ–Є—Б—В–µ—А—Б—В–≤–Њ –Њ–±–Њ—А–Њ–љ—Л" вЖТ captured as just "–Ь–Є–љ–Є—Б—В–µ—А—Б—В–≤–Њ" (single, not entity). After: anchor + extend-right algorithm captures "–Ь–Є–љ–Є—Б—В–µ—А—Б—В–≤–Њ –Њ–±–Њ—А–Њ–љ—Л" as a 2-word entity, stopping at stop-words/punctuation. **Direct fix for AC3.**
- **[POSITIVE] Numeric morphology added to `_match_in_text`.** New 4th level: regex-based stemmer + small RU/EN dict (`_MORPH_MAP`) normalizes "–і—А–Њ–љ–∞/–і—А–Њ–љ–Њ–≤/–і—А–Њ–љ—Л" вЖТ "–і—А–Њ–љ", "drone/drones" вЖТ "drone", "—Б–±–Є—В–Њ/—Б–±–Є—В—Л/—Б–±–Є—В" вЖТ "—Б–±–Є—В". Score 90 if numbers match, 85 if they differ (stem-equal is the strong signal). **Direct fix for AC4.**
- **[POSITIVE] Single short capitalized no longer leaks.** AC2 ("Python", "–Ь–Є–љ–Є—Б—В–µ—А—Б—В–≤–Њ" as single words not extracted) satisfied. `FACT_RE_ENTITY_SOLO` is currently set to `(?!)` (never-matches) as the conservative choice; can be re-enabled later with proper length threshold.
- **[POSITIVE] No new dependencies.** Patch uses only stdlib (`re`, no new packages). `rapidfuzz` already imported.
- **[POSITIVE] No public API change.** Signatures of `_extract_facts(text, max_facts=8)`, `_match_in_text(fact, text)`, `_is_negated(fact, text)`, `deep_research()`, `deep_search()` all unchanged. Two new public helpers: `_morph_stem(word)` and `_normalize_num_unit(phrase)`. These are intentionally exported (underscore prefix = module-private but testable) and not used by any external caller.

### вЪ†пЄП RESIDUAL

- **[LOW] `_MORPH_MAP` is a hand-curated dict, not a stemmer.** It covers the 7 most common cases in news content (–і—А–Њ–љ, —Б–±–Є—В, –Њ–±–љ–∞—А—Г–ґ, –њ–µ—А–µ—Е–≤–∞—З, —Г–љ–Є—З—В–Њ–ґ, —З–∞—Б, –Љ–Є–љ) + 5 EN plurals. New content with unfamiliar words falls back to naive suffix stripping (`-–Њ–≤`, `-—Л`, `-s`, etc.) which is approximate. **Acceptable for v0.8.2** вАФ measurable AC4 hit rate is the priority. Future: plug in `pymorphy3` or similar proper Russian stemmer.
- **[LOW] `_normalize_num_unit` greedy match.** `NUM_UNIT_RE` uses `re.search` on the phrase, so for "123 –і—А–Њ–љ–∞ –Є 5 —Б–±–Є—В–Њ" it returns the FIRST number+word pair ("123 –і—А–Њ–љ–∞"). For multi-number facts (rare), this loses info. **Acceptable for v0.8.2** вАФ `verify_sources` extracts facts one at a time and feeds them to `_match_in_text` individually.
- **[LOW] `FACT_RE_ENTITY` greedy extension can over-extend.** Algorithm extends entity rightward as long as next word is вЙ•3 chars and not a stop word. For "–Ь–Є–љ–Є—Б—В–µ—А—Б—В–≤–Њ –Њ–±–Њ—А–Њ–љ—Л –≤—Л—Б—В—Г–њ–Є–ї–Њ —Б –Ј–∞—П–≤–ї–µ–љ–Є–µ–Љ" вЖТ captures "–Ь–Є–љ–Є—Б—В–µ—А—Б—В–≤–Њ –Њ–±–Њ—А–Њ–љ—Л –≤—Л—Б—В—Г–њ–Є–ї–Њ" (3 words). The 3rd word "–≤—Л—Б—В—Г–њ–Є–ї–Њ" is a verb, not part of the entity name. **Mitigated by max 4-word cap.** Could be improved with a small POS-tag heuristic, deferred.
- **[LOW] New `STOP_CAPS` additions ("—Б–µ–≥–Њ–і–љ—П", "–≤—З–µ—А–∞", "—Б–µ–є—З–∞—Б", "—В–µ–њ–µ—А—М") overlap with date-extraction intent.** These words sometimes precede dates ("–≤—З–µ—А–∞, 5 –Є—О–љ—П"). The current code handles this fine because the date extractor runs BEFORE the entity extractor; "5 –Є—О–љ—П" is already in `seen` when entity extractor runs. **No regression risk.**
- **[NOTE] `_add()` now mutates `seen` and `facts` mid-iteration.** The new reverse-substring branch does `seen.discard(old)` and `facts.remove(old)` after collecting `to_remove` in a separate list. This is correct (Python 3 `list.remove` is O(n) but `facts` is bounded by `max_facts=8`, so 8√Ч8 = 64 comparisons max in the worst case). No concurrency issues since `_extract_facts` is single-threaded.
