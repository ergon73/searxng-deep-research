## Findings

### ✅ POSITIVE — what we fixed

- **[POSITIVE] Verdict enum now matches core semantics.** Before: LLM used `MATCH/NO MATCH` (binary), but core uses `SUPPORTS/REFUTES/INSUFFICIENT/CONFLICTING`. After: LLM prompt asks for 3-value enum, schema enforces it via `json_schema.strict: True` with `enum: [SUPPORTS, REFUTES, INSUFFICIENT]`. **LLM can now signal refutation** (it could only signal "match/mismatch" before).
- **[POSITIVE] Strict json_schema replaces loose json_object.** Before: `response_format: {"type": "json_object"}` — model could return any JSON. After: `response_format.json_schema.strict: True` — OpenRouter guarantees the response matches the schema. **Defensive fallback** still parses JSON in case the model misbehaves.
- **[POSITIVE] LLM errors are now structured, not silently swallowed.** Before: 401/timeout/JSON-parse all became generic "LLM error: <msg>" in `reasoning` field; `socket.timeout` was uncaught at all (uncaught exception). After: all errors return `{"verdict": None, "llm_error": "<Type>: <msg>"}` with `llm_verified=False`. `verify_sources()` propagates `llm_error` to its return dict.
- **[POSITIVE] `verify_fact` (semantically broken) removed.** Before: `verify_fact(fact, top1, candidate)` asked "are A and B about the same fact?" with prompt "is fact from A confirmed by B?" — internal inconsistency. After: method deleted; new `verify_claim_against_evidence(claim, evidence_blocks)` is the single, semantically clear entry point.
- **[POSITIVE] No new dependencies.** Patch uses only stdlib (`socket` for timeout — already stdlib). No new packages.
- **[POSITIVE] Backward-compatible public API for callers.** `verify_sources()` return dict extended with `llm_error: None` (additive, not breaking). All 5 existing `test_verify_schema.py` tests pass unchanged except `test_no_llm_when_disabled` which now also asserts `llm_error is None`.
- **[POSITIVE] Defensive verdict normalization.** `_normalize_verdict()` strips whitespace, uppercases, and maps legacy `MATCH → SUPPORTS`, `NO MATCH → INSUFFICIENT`. Unknown values → `INSUFFICIENT` (conservative default). Test `test_match_enum_is_removed` locks this in.

### ⚠️ RESIDUAL

- **[LOW] `_post()` does its own retry loop, but `verify_facts_batch` calls it once.** The retry is for transient HTTP errors. The error path is now uniform (raise → caught at call site). This is fine for v0.8.2.
- **[LOW] `verify_claim_against_evidence` is a thin wrapper around `verify_facts_batch([claim], ...)`** — single-fact batch. Slight overhead vs. dedicated single-fact schema, but reuses the same code path which is what DR §14 wanted ("переписать отдельно" → "or implement as separate method"). Acceptable.
- **[LOW] `max_tokens = 200 + 80 * len(facts)`** in batch — accounts for `verdict` + `reasoning` + `source_urls` per fact. For 10 facts: 1000 tokens. Reasonable. If user bumps `max_facts=20` → 1800 tokens, still within most model limits.
- **[LOW] `socket.timeout` aliased to `TimeoutError` since Python 3.10** (in `urllib.error` context). Ruff warns UP041 ("use builtin TimeoutError"). Kept `socket.timeout` for explicit intent in `except` clauses — both catch the same exception. Cosmetic.
- **[NOTE] The `CONFLICTING` verdict is local-only, not LLM-returnable.** `verify_sources()` derives `CONFLICTING` from "has both supporting and refuting sources for the same fact" — this is a higher-level synthesis that the LLM doesn't do. The LLM gives atomic verdicts per fact; core glues them. The test `test_refutation_in_other_source` exercises this path. No change needed.
- **[NOTE] `URLError` is the parent of `HTTPError` in urllib** — listing both in `except` is redundant. Kept for explicit intent (HTTP errors vs. lower-level network errors). Cosmetic.

### Not changed (still vulnerable / broken, deferred)

- **[MEDIUM] Default LLM model is still `meta-llama/llama-3.1-8b-instruct:free` (DR §12).** Per DR: user should set `LLM_MODEL=minimax/minimax-m3` explicitly. Not part of Phase 4 scope. **No action.**
- **[MEDIUM] `prompt` doesn't yet use the schema in its textual instructions.** The schema is enforced server-side, but the user-facing prompt still says `verdict: "SUPPORTS"|"REFUTES"|"INSUFFICIENT"` in plain text. Could be removed since schema is the source of truth, but doesn't hurt. **No action.**
- **[LOW] LLM latency not measured inside `_post()`.** The integration in `verify_sources()` measures `time.time()` around the LLM call. Same as before. **No change needed.**
